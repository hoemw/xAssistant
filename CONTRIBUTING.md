# 贡献指南

感谢您对 xAssistant 项目的贡献！

## Git 提交规范

本项目要求所有 Git 提交信息使用中文，并遵循以下规范：

### 提交信息格式

提交信息应包含以下部分：

```
<类型>: <简短描述>

<详细描述>（可选）

<页脚>（可选）
```

### 提交类型

- **feat**: 新功能
- **fix**: 修复 Bug
- **docs**: 文档更新
- **style**: 代码格式调整（不影响功能）
- **refactor**: 重构代码
- **perf**: 性能优化
- **test**: 添加或修改测试
- **chore**: 构建过程或辅助工具的变动
- **build**: 构建系统或外部依赖的变动
- **ci**: CI 配置文件和脚本的变动
- **revert**: 回退之前的提交

### 提交示例

#### 示例 1: 新功能
```
feat: 添加快捷光照工具

实现了一键快速添加常用光源的功能：
- 支持添加定向光、点光源、聚光灯
- 自动设置合理的默认参数
- 集成到工具菜单中
```

#### 示例 2: Bug修复
```
fix: 修复属性修改工具的空指针异常

在选择集为空时调用属性修改工具会导致崩溃
现在添加了空值检查并给出友好提示
```

#### 示例 3: 文档更新
```
docs: 更新安装配置说明

补充了 DeepSeek API 的配置步骤
添加了常见问题解答章节
```

#### 示例 4: 性能优化
```
perf: 优化场景上下文采集性能

将 Actor 遍历改为增量更新
减少了 50% 的上下文采集时间
```

### 提交信息规范要求

1. **使用中文**: 所有提交信息必须使用中文
2. **简短描述**: 第一行简短描述不超过 50 个字符
3. **详细描述**: 如有需要，空一行后添加详细描述
4. **条理清晰**: 使用列表或分点说明具体变更
5. **说明原因**: 重要变更应说明"为什么"而不仅是"是什么"

### 不良示例

❌ 不符合规范的提交：
```
update files
修改了一些东西
fix bug
添加新功能
```

✅ 符合规范的提交：
```
feat: 添加批量重命名工具

实现了批量重命名选中 Actor 的功能
支持自定义前缀和起始序号
```

## 代码提交流程

1. 创建功能分支
   ```bash
   git checkout -b feature/功能名称
   ```

2. 开发并提交代码
   ```bash
   git add 文件名
   git commit -m "feat: 功能描述"
   ```

3. 推送到远程仓库
   ```bash
   git push origin feature/功能名称
   ```

4. 创建 Pull Request

## 代码规范

- 遵循虚幻引擎代码规范
- 使用中英文双语注释（重要逻辑必须有中文说明）
- 保持代码整洁，避免冗余
- 为新功能添加相应的文档

## 文档贡献

- 文档应清晰易懂
- 提供必要的示例代码
- 保持文档与代码同步更新
