# 上下文记忆体 设计说明（v0.1 草案）

目标与约束

- 完全离线、本地单机运行；不依赖外网与云服务。
- 中文优先，兼顾英文；输入输出与检索对中文友好。
- 与现有插件架构无缝集成，尽量复用既有接口与风格。
- 安全可控：可审计、可清理、可导出；默认最小权限。

总体思路

- 分层记忆 → 多路检索 → 重排压缩 → 规划执行 → 反思写回。
- 检索作为“非参数化记忆”，与生成过程紧密耦合（RAG范式）。

一、分层记忆模型

1) 短期/工作记忆（STM）

- 最近K轮对话、活跃对象、临时变量；内存为主，必要时落盘。

2) 情景记忆（Episodic）

- 会话或任务级决策、约定、错误→修复、关键链接；需要时间与重要度。

3) 长期语义记忆（Semantic）

- 稳定事实与文档切片；向量索引便于语义相似检索。

二、本地存储与索引

- 结构化/全文：SQLite + FTS5（单文件、可移植、零服务）。
- 向量索引：优先 FAISS-CPU（纯本地），可替换 Chroma（本地嵌入式）。
- 嵌入模型：bge-small-zh-v1.5（中英双语，中文效果佳）。未来可替换为 gguf 本地推理或现有 Embedding 服务代理到本地HTTP。

三、数据结构（草案）

- Tables（SQLite）：
  - messages(id, session_id, role, content, ts)
  - episodic(id, session_id, title, summary, content, importance, ts, tags)
  - docs(id, source, chunk_id, text, ts, tags)
- Index：
  - FTS5(messages_fts(content), episodic_fts(content, summary), docs_fts(text))
  - 向量：docs.vec / episodic.vec（FAISS 索引文件），或 SQLite BLOB 存储 + 内存扫描 PoC

四、检索与重排

- 多路召回：
  - FTS/BM25：messages_fts、episodic_fts、docs_fts
  - 向量Top-K：对 query embedding 检索 episodic.vec + docs.vec
- 融合重排：语义分数 + 词匹配 + 时间衰减 + 重要度（归一化加权）。
- 上下文压缩：对Top-K证据做摘要（长度/覆盖率/去冗余）以适配上下文窗口。

五、规划与执行

- 策略：
  - ReAct：推理-行动交替，适合“工具执行 + 检索问答”。
  - Tree of Thoughts：复杂任务多路径探索与自评（控制搜索预算与深度）。
  - Reflexion：执行结果与人类反馈沉淀“反思记忆”，用于后续修正。

六、离线最小可行（PoC）方案

- A 方案（零外部原生库，最快验证）：
  - SQLite+FTS5 作为主检索；向量以 SQLite BLOB 存储，内存暴力Top-K扫描（适度数据量足够）。
  - 验证流程、API、数据模型与评估闭环。
- B 方案（集成向量库，性能更好）：
  - 引入 FAISS-CPU，构建本地索引文件；需要在 Build.cs 配置第三方库与打包。
- 建议：先 A 再 B（迭代切换仅在“向量读写适配层”）。

七、与 UE 快捷工具（第一期）联动

- 统一“命令面板”入口：中文可检索工具名/描述，执行前可选预览与确认。
- 覆盖范围：SelectionTools / ViewportTools / ActorTools（放置/移动/重命名/删除）/ SystemTools（遵守 bSkipConfirmForConsoleCommands）/ QueryTools。
- 输出格式：工具计划（ToolId+参数）与证据摘要并行呈现（可审计）。

八、评估与指标

- 信息命中率（检索Top-K中含答案证据的比例）。
- 回答正确率/有用性（基于小型标注集或对照问答）。
- 长期一致性（跨会话的设定与偏好保持度）。
- 性能：检索时延、上下文压缩耗时、端到端响应时延。

九、安全与隐私

- 分区：私有/会话/团队标签；敏感字段可选脱敏/摘要化存储。
- 工具危险级与二次确认；一键清除与整体导出（SQLite+索引文件）。
- 日志审计：记录证据/工具调用轨迹；可开关与留存周期。

十、实施与里程碑

- M1（PoC-功能闭环）：
  - 存储与数据结构、FTS检索、简易向量扫描、重排与压缩、命令面板入口。
- M2（性能与策略）：
  - 接入 FAISS-CPU、策略开关（ReAct/ToT/Reflexion）、可配置搜索预算。
- M3（安全与可运维）：
  - 导出/清理、权限标签、可审计日志、评估脚本与回归基线。
